## Таблица 2. Права файлов после Задания 2 (после chmod)

| Файл | Установленные права | Команда chmod |
|------|---------------------|---------------|
| script.sh | -rwxr-xr-x (755) | chmod +x script.sh |
| secret.txt | -rw------- (600) | chmod 600 secret.txt |
| readme.md | -r--r--r-- (444) | chmod 444 readme.md |
| project_folder/ | drwx------ (700) | chmod 700 project_folder |

**Объяснение:**
- `chmod +x script.sh` - добавляет право выполнения для всех
- `chmod 600 secret.txt` - только владелец может читать и писать
- `chmod 444 readme.md` - все могут только читать, никто не может писать
- `chmod 700 project_folder` - только владелец имеет полный доступ к папке

## Ответ на вопрос

**Почему для файла database.conf нужны права 600, а не 644?**

Файл database.conf содержит чувствительные данные, такие как пароли базы данных, хосты, порты и другие учётные данные подключения. Если установить права 644, то любой пользователь системы (в группе или вообще любой) сможет прочитать этот файл и получить доступ к паролям, что создаёт серьёзную уязвимость безопасности. Права 600 (rw-------) обеспечивают, что только владелец файла (обычно администратор или главный разработчик) может читать и редактировать конфиг, защищая критически важные секреты приложения.
